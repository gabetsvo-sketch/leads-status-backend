# RISK_MAP — leads-status/backend

Красные зоны, владельцы, симптомы и стоп-условия (§9, §17–19 стандарта VIBE-CODE v0.01.001).
Режим проекта: **БОЕВОЙ ПРОДУКТ** (обслуживает iOS-приложение, подключён к amoCRM, реальные данные клиентов).

| Зона | Чем опасна | Владелец решения | Стоп-условие |
|---|---|---|---|
| **Запись в amoCRM** (смена статуса сделки, задачи, причины отказа) через очередь `crm_actions` → CRM-воркер | необратимое изменение боевой сделки клиента; авто-задачи Salesbot (инцидент Mary Land 2026-05-02) | Владимир | запись в CRM — ТОЛЬКО по явному действию Владимира в приложении; никакой авто-записи |
| **Отправка сообщений клиентам** | сообщение нельзя «отозвать»; репутация | Владимир | backend сам клиентам НЕ шлёт; всё через ручное одобрение Владимира (no auto-send) |
| **Движок стиля** (`/style-runtime/v1/draft`) | плохой/небезопасный черновик клиенту | Владимир | все черновики `manual_review_only`; safety-gate; не отправлять без проверки человеком |
| **Токены** `WIDGET_TOKEN` / `INTERNAL_TOKEN` / `OFFICE_TOKEN` | утечка → посторонний доступ к API сделок | Владимир | только в env Render (`sync:false`), не в коде/логах/git; не печатать в ответах |
| **APNs-пуши** (на iPhone Владимира) | спам/потеря уведомлений о заявках | Владимир | ключи `APNS_*` в env Render; не рассылать пуши пачками |
| **Состояние = JSON-файлы на диске Render** (`leads_inbox`, `tasks_today`, `devices_registry`, `crm_actions`, `sent_events`, `crm_catalog`) | потеря файла = потеря состояния; нет реляционной БД и автобэкапа | Владимир | не удалять/перезаписывать файлы состояния вслепую; изменения формата — совместимо |
| **Деплой на Render** | боевой сервис iOS-приложения | Владимир | деплой под защитой CI (verify → deploy); обход — только осознанный `force_deploy` |

## Чего в проекте НЕТ (зоны не применимы)
- **Платежи/тарифы/балансы** — отсутствуют. Денежной зоны в backend нет.
- **Реляционная БД / миграции** — нет; состояние в JSON-файлах (см. выше).
- **Авторизация пользователей-клиентов** — нет; доступ к API только по bearer-токенам (один оператор — Владимир).

## Постоянные правила безопасности
- Никаких отправок клиентам, записей в CRM, авто-рассылок без ручного одобрения Владимира.
- Не выдавать заглушку/недоступность за факт (честный статус).
- Изменения красных зон — только с явным подтверждением Владимира (§9).
