# Клиент задаёт конкретный вопрос

## Когда использовать

Использовать, когда последним сообщением клиент задал конкретный вопрос: цена, рассрочка, актуальность, просмотр, документы, доходность.

## Статус

`candidate`. V2 pack: сценарий уточнён, примеры и CTA обезличены; raw-сообщения не копируются.

## Что реально видно в сообщениях

- Когда клиент пишет первым с вопросом, ответ начинается с прямого ответа, а не с презентации.
- После ответа добавляется один следующий шаг: уточнить, показать, созвониться, отправить материал.
- Если вопрос требует данных по объекту — не подставлять цифры без текущего источника.

## Паттерн ответа

1. Сначала прямой ответ на вопрос клиента.
2. Затем короткое пояснение, если без него ответ может быть непонятен.
3. Затем один следующий шаг: показать вариант, уточнить параметр, созвониться, отправить документ/подборку.

## Что избегать дополнительно

- Не уходить от вопроса в общую презентацию объекта.
- Не отвечать новым длинным подбором, если клиент спросил конкретную вещь.
- Не подставлять цены/рассрочку/актуальность без текущего источника.

## Runtime-инструкция

1. Определить сценарий и стадию диалога.
2. Взять 2–4 sanitized паттерна из этого pack.
3. Не копировать паттерн дословно; повторить структуру: спокойный тон → практический ответ/объяснение → один следующий шаг.
4. Не добавлять конкретные цены, названия проектов, проценты, обещания доходности или юридические гарантии без проверенного источника из текущего контекста.

## Что избегать

- Давление и искусственная срочность.
- Обещания гарантированной доходности/результата.
- Длинный поток сообщений без вопроса или следующего шага.
- Копирование raw-фраз с именами, телефонами, URL, конкретными цифрами или названиями проектов.
